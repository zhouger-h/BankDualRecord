# 双录控件编译说明

## 方案一：GitHub Actions 自动编译（推荐）

### 步骤

1. **将代码推送到 GitHub 仓库**
   ```bash
   # 在 dual-record-system 目录下
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/BankDualRecord.git
   git push -u origin main
   ```

2. **触发自动编译**
   - 推送到 main 分支会自动触发编译
   - 或进入 GitHub 仓库 → Actions → Build DualRecord Control → Run workflow

3. **下载编译结果**
   - 编译完成后，进入 Actions 页面
   - 点击最新的 workflow run
   - 在 Artifacts 区域下载：
     - `BankDualRecord-Setup-Windows-x86` — Windows 安装包
     - `BankDualRecord-Portable-Windows-x86` — 绿色便携版
     - `BankDualRecord-Kylin-x86_64` — 麒麟可执行文件

### 编译环境

| 平台 | 环境 |
|------|------|
| Windows | windows-2019 + Qt 5.15.2 (MinGW 32-bit) + NSIS 3.x |
| 麒麟 | ubuntu:20.04 + Qt5 + GCC |

## 方案二：本地编译

### Windows 32位

**前置要求：**
- Qt 5.15.2 (MinGW 8.1.0 32-bit)
- NSIS 3.x (用于打包安装程序)

**编译步骤：**

```bat
REM 打开 Qt 5.15.2 (MinGW 8.1.0 32-bit) 命令行
cd dual-record-system

REM 1. 生成 Makefile
qmake DualRecordService.pro

REM 2. 编译
mingw32-make release -j4

REM 3. 部署 Qt 依赖
mkdir deploy
copy release\DualRecordService.exe deploy\
windeployqt --release --force deploy\DualRecordService.exe

REM 4. 打包安装程序（可选）
cd installer
"C:\Program Files (x86)\NSIS\makensis.exe" install_windows.nsi
```

**输出文件：**
- `release/DualRecordService.exe` — 主程序
- `installer/BankDualRecord_Setup_v1.0.0_x86.exe` — 安装包

### 银河麒麟 / 信创

**前置要求：**
```bash
sudo apt-get install qt5-default qtbase5-dev qtmultimedia5-dev \
    libqt5multimedia5-plugins libssh2-1-dev libssl-dev
```

**编译步骤：**

```bash
cd dual-record-system
qmake DualRecordService.pro
make -j$(nproc)

# 输出：DualRecordService（可执行文件）
```

## 内网部署步骤

### Windows 内网

1. 将 `BankDualRecord_Setup_v1.0.0_x86.exe` 拷贝到内网
2. 以管理员身份运行安装
3. 首次启动配置：
   - 影像缓存路径（如 `D:\DualRecord\Cache`）
   - FTP/SFTP 服务器地址
   - TCP 监听端口（默认 9527）

### 麒麟内网

1. 将 `DualRecordService` 和 `install_kylin.sh` 拷贝到内网
2. 执行安装：
   ```bash
   sudo bash install_kylin.sh
   ```
3. 首次启动自动弹出配置向导

## 常见问题

### Q: GitHub Actions 编译失败？
A: 检查 `.github/workflows/build.yml` 中的 Qt 版本和架构是否匹配

### Q: 内网无法连接 GitHub？
A: 使用方案二本地编译，或在外网编译好后将安装包拷贝入内网

### Q: 缺少 DLL 无法运行？
A: 确保使用 `windeployqt` 部署了所有 Qt 依赖库

### Q: 摄像头无法识别？
A: 检查是否安装了摄像头驱动，以及是否有权限访问
